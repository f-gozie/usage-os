Three ways to read the same day

Timeline — pick a direction

One real morning: deep work bouncing between usageos and nudge, then comms, research and a break — none of which carry a project. Same data, segmented three ways. Which one reads true?

A · By kind of work
Blocks are runs of the same context; the project split shows inside each block. Project-hopping never fragments it, and it matches the dial (which colours by context). Optimises for “what kind of work,” with project as texture.
B · By project
Blocks are runs of the same project. Truer to “I worked on two projects,” but the interleaved morning fragments into many short rows — and all your comms/research/breaks collapse into long “No project” stretches (we can't attribute them). Optimises for “time per project.”
C · Both axes at once
Two aligned lanes: kind of work on top (coloured), project below (labelled — projects are named, not coloured, since there can be many). You read both independently. Most honest that they're separate, but it's an overview, not a place to read detail.
