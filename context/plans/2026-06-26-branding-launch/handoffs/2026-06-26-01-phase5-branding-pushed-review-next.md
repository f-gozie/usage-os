## 2. Key decisions (all locked this session; see `plan.md` decision table)
- **Mark = "Contexts"** — the day-dial: faint track + 3 category arcs (Work blue `--c-deep`, Browsing red `--c-research`, Messaging gold `--c-comms`), midnight gap at top. Chosen over 11 other logo directions.
- **Wordmark = dial-O**: the Contexts dial **is** the letter O in `USAGE◑S`. **It stands ALONE — never paired with a separate dial icon** (two dials side-by-side reads as redundant — owner feedback). The standalone mark is only for icon-only spots (app icon, favicon, tray).
- **Dial-O geometry FROZEN:** diameter **1.02em** · baseline nudge **0.08em** · ring weight **17** (of 100 viewBox) · side space **−0.05em**.
- **Dial-O only renders large (≥~22px):** the big app header (`AppShell`, 36px), landing, About. **Tiny chrome (titlebar / glance / onboarding header) stays plain text** — the dial-O muddies ≤16px (confirmed live; the owner rejected it in the titlebar).
- **Name:** `USAGEOS` (poster caps) in logo/headings; `UsageOS` (one word, never "Usage OS") in prose. Technical ids stay: repo `usage_os`, crate/binary `usage-os`, bundle `com.favour.usage-os`. (Dev dock shows "usage-os"; a release `.app` shows "UsageOS" — `productName` is correct.)
- **App-icon spec:** bone tile (`#EEEBE1`), arc weight 9, padding 15, gap 18°, track off, no now-hand (static). Colour down to ~24px, mono template below.
- **Tray:** mono **template** with a **live now-hand** pointing at the current hour (a clock, not a HUD).
- **Splash:** no blocking splash. Motion = **"H · wordmark builds"** as the onboarding Welcome opening beat (first run only); plus the existing day-load draw.
- **Brand colour:** monochrome (ink + paper) base, **blue the only chrome accent**; the 3 category colours live **only inside the dial**; **no green** on brand surfaces; no big colour blocks (a "bold" loud-blue landing was rejected).
- **Positioning:** lead with **screen-time, honestly** (per the tweet-voice memory).
- **Landing:** **Astro** static site → **Cloudflare Pages**; the **"Product"** direction (framed real app window as hero) chosen over Calm / Monument / Bold / template passes.
- **Demo data:** real app names are fine; **eyemark removed** from the demo catalog (owner's other project).
- **License: MIT** (confirmed correct for a free, local, auditable app).
- **Motion tech:** pure CSS/SVG in the app; GSAP only-if-needed on the landing (not used yet).

## 8. Important context / gotchas
- **Driving the app for screenshots:** `tauri dev` runs the raw binary `usage-os`, which is **not a registered app**, so **computer-use can't control it**. Use **`cliclick`** (installed) + **`screencapture -R 166,36,1180,940`** (window bounds via `osascript ... get {position,size} of window 1`). **Focus races** with the user's Claude window are common — re-`osascript activate` right before each capture, and for tab switches it's most reliable to **let the owner click the tabs** while you only activate+capture.
- **Demo data:** `cargo run --features perf --bin seed_db -- --demo --days 8 --end $(date -v0H -v0M -v0S +%s) --out /tmp/x.db --force` → blocky single-category days, this-week dates. To load: quit app, set `onboarding_completed=true` + `theme=paper` in the seed's `settings`, `PRAGMA wal_checkpoint(TRUNCATE)`, copy over `~/Library/Application Support/com.favour.usage-os/usage.db`, relaunch.
- **DB safety:** the owner's real DB was backed up to **`/tmp/usageos-realbak/`** (usage.db + -wal + -shm, 2094 rows) and **restored exactly** at session end. That backup is safe to delete. **TablePlus** also holds the DB open (a read viewer) — owner may want to reconnect it.
- **App is currently STOPPED.** Relaunch with `npm run tauri dev` (rebuilds ~15–40s). The owner's original terminal `tauri dev` was killed during the screenshot dance.
- Fonts (Anton/Jost) are bundled locally everywhere (hard rule 1) — landing too.
