# Phase 1.2a — Capture seam (impl plan, as-built)

_Reconstructed from PR #11 (the plan-mode scratch was overwritten; this is the as-built record, which matches the approved plan). See also the decision record and the handoff for this phase._

Introduces the `capture/` boundary — the second step of Phase 1. All capture goes through a mockable `CaptureSource` trait (hard rule 5, D22/D29), so the app builds and tests without macOS or permissions. This is the **CI-testable spine**; the event-driven macOS impl is a deliberate **on-device-verified follow-up (1.2b)** — CI compiles native code but can't grant TCC / run a GUI. Records **D32**.

## What landed
- **`capture/mod.rs`** — `CaptureSource` trait, `FocusEvent` (owned/`Send`, marshaled per D29), `process_focus_event` (applies **D8** exclusions: `Exclude` drops the event, `Private` records time but omits title/url — R58), `run` runner draining the channel, the health counter behind `get_watcher_status`, and `default_source()` (the single switch 1.2b flips to the macOS impl).
- **`capture/fake.rs`** — `FakeCapture`: seeded events; the mock that exercises the whole spine headless (hard rule 5 / capture standard C4).
- **`capture/polling.rs`** — `PollingCapture`: the v0.1.0 `active-win-pos-rs` + `user-idle` logic moved behind the trait (interim production source, no regression; kept as the non-macOS fallback). **App name + idle only — not titles:** `active-win-pos-rs` titles come from `CGWindowName` (Screen Recording, capture standard C1, the permission we refuse). Real titles + url/cwd arrive via AX in 1.2b.
- **`db.rs`** — `log_focus` (coalescing write carrying `is_private`/`url`/`site` via `insert_event`); `log_activity` kept as a thin wrapper so existing coalescing tests are unchanged. `now_unix` made `pub`.
- **`watcher.rs` deleted** — role folded into `capture::run`. `lib.rs` setup spawns `capture::run(db_conn, default_source())`; `get_watcher_status` reads `capture::get_error_count`. The `WatcherStatus` command name is kept (renaming would churn bindings).

## IPC impact
**None.** No new commands; `FocusEvent`/`CaptureSource` are internal. `bindings.ts` is byte-identical (freshness gate passes).

## Behavior note
Between this PR and 1.2b, capture records **apps (+ idle), not titles**. That's honest degraded mode — the v0.1.0 path "captured" titles that were always empty anyway (the `CGWindowName` bug). Exclusions (D8) on app name work now; site/title/project enrichment lands with the AX signals in 1.2b.

## Out of scope — 1.2b (next PR, on-device verified)
Event-driven macOS `CaptureSource`: NSWorkspace + per-PID `AXObserver` on Tauri's main run loop (port `spikes/ax-observer`); browser URL + incognito skip (port `spikes/browser-url`); terminal cwd + front-tab pid selection (port `spikes/proc-cwd`); then site parsing + project inference (`spikes/project-inference` → `resolve_or_create_project`). Flips `default_source()`.
