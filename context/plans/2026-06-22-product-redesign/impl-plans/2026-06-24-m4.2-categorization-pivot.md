# M4.2 ‚Äî Categorization closeout: embeddings spiked & shelved, app-metadata suggestions + relatable defaults

_As-built, 2026-06-24. Branch `phase2/m4-settings`. Decision: **D47**. Closes the last open Phase-2 item ("embedding-based categorization + corrections memory")._

## Why
Phase 2 planned embedding-based categorization (D10/D26). We spiked it before building (the project's spike-before-reliance habit) and it **failed on accuracy** ‚Äî see `spikes/embeddings/README.md`. The mechanism is sound (callable from Rust, off-thread, 512-d, ~3 ms, no Apple Intelligence) but similarity over app/window-title strings scores **below the majority-class baseline** (app-name LOO 39% / centroid 41% / title 60% vs 43%). Brand names ("Cursor", "Steam") are out-of-vocabulary noise to a sentence model. So we did **not** build it.

## What shipped instead (D47)
1. **Rules stay primary** + **D44's Uncategorized‚ÜíAssign flow is the corrections memory** (a correction ‚Üí a retroactive `process` rule). Nothing new needed there.
2. **`LSApplicationCategoryType` ‚Üí category suggestions** ‚Äî deterministic, conservative, abstaining. A one-click "sort into X" in the Uncategorized list. Suggestion only; the rules engine still assigns.
3. **Relatable default taxonomy** ‚Äî Work ¬∑ Browsing ¬∑ Messaging ¬∑ Entertainment ¬∑ **Personal** (5th canonical, theme-aware green token). Fresh-installs-only migration; existing DBs untouched.
