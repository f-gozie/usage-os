# Impl-plan ‚Äî M4.1: app icons (app-wide) + categories/rules UX + Timeline memory

_2026-06-24. Branch `phase2/m4-settings`. Follows the M4 Settings shell ([D42](../../../decisions.md)). Driven by on-device dogfooding feedback: the categories model was opaque, there was no validation, input was blind free-text, the UI looked bare without icons, and the app OOM'd at ~2.13 GB._

## Goals (in priority order)

1. **App icons, everywhere an app is named** ‚Äî the immediate want. A shared `AppIcon` primitive resolved from the user's installed apps, used in the Timeline switch rows + run summary, the Day legend, and Settings. Graceful monogram/colour fallback for names that don't resolve.
2. **Categories & rules, made legible** ‚Äî an Unsorted-apps section, an app-picker (no blind typing), duplicate/conflict validation, a live match preview, plain-language copy. (Approved mockup: `design/settings-categories.html`.)
3. **Timeline memory** ‚Äî virtualize the app-switch list; strong evidence it's the real driver of the 2.13 GB (webkit RSS spiked to 290 MB rendering it; Rust stayed flat ~100 MB).

Ship in that order ‚Äî icons are a vertical slice that lands value first.

## Spike results (de-risked, on this Mac)

- Enumerating `/Applications`, `/System/Applications`, `~/Applications`, **and PWA dirs** (`~/Applications/Chrome Apps.localized/`, `Brave Browser Apps.localized/`) needs **no permission**. (nudge lives there ‚Äî that's why it "had no icon".)
- Every sampled app ships a **loose `.icns`**; `sips -s format png -Z 64` yields a correct **~5‚Äì15 KB** 64px PNG, offline, instant. 27 icons extracted to `design/assets/app-icons/` total 380 KB.
- `usage-os` (the dev binary) has no installed bundle, but **our own icon lives at `src-tauri/icons/`** ‚Äî the shipped app resolves to it.
- `NSWorkspace.iconForFile` (objc2) also works universally but returns full-res (1‚Äì2 MB) and needs NSImage drawing ‚Äî kept only as a noted fallback; not used in v1.

## Decisions baked in

- **Icon extraction = loose `.icns` + `sips`, cached to disk.** Zero new crates; the app already shells `git` (`enrich/project.rs`). Behind a mockable `apps` trait (hard rule 5). No new TCC, no network (hard rule 1). NSWorkspace/pure-Rust `icns` decode noted as future hardening.
- **Unsorted scope = all time tracked** (retention-bounded), ranked by total secs, floored (<~120 s hidden), collapsible. Rationale: rules are retroactive (reprocess re-sorts all history), so sorting an app fixes every past day ‚Äî all-time is the honest scope. (Confirmed with user.)
- **Conflict detection is a UI affordance, computed frontend-side** from the `rules` list (substring, first-by-id ‚Äî mirrors `find_category` for process rules). The engine still enforces first-match at `reprocess_logs`; this only *warns*. No new chatty backend call.
- **Icons delivered as base64 PNG inline** in `list_installed_apps()`, resolved into a name‚ÜídataURI map loaded **once** at app start (module-level cache). Shared `src` ‚áí the webview decodes one copy per unique icon, not per row ‚Äî safe even in the long Timeline list.

## Phase A ‚Äî App icons in the app

### Backend (`src-tauri/src/`)
- **New `apps/mod.rs`** (module + `AppCatalog` trait, mockable):
  - `InstalledApp { name: String, bundle_id: Option<String>, icon_png_b64: Option<String> }` (`specta::Type`).
  - `list_installed() -> Vec<InstalledApp>`: enumerate the app dirs (recurse one level into `*.localized` PWA folders); name = bundle stem; resolve `.icns` (CFBundleIconFile via `defaults read`, else first `*.icns`); `sips`‚Üí64px PNG; cache to `<appsupport>/icon-cache/<bundle-stem>.png` keyed by `.icns` mtime; base64 inline. No `unwrap/expect/panic` ‚Äî every failure ‚Üí `icon_png_b64: None` (UI monograms).
  - `FakeCatalog` for tests.
- **`lib.rs`**: `#[tauri::command] list_installed_apps() -> Result<Vec<InstalledApp>, AppError>`; register in the `collect_commands!` Builder.
- Regenerate bindings: `cargo test export_bindings` (freshness gate: `git diff --exit-code src/bindings.ts`).
- Tests: catalog parsing against a fixture .app tree; cache hit/miss; mock returns deterministic icons.

### Frontend (`src/`)
- `lib/tauri.ts`: `listInstalledApps()` wrapper + `InstalledApp` re-export.
- `lib/appIcons.ts`: module-level memoized `getIconMap(): Promise<Map<string,string>>` (process-name-normalized ‚Üí dataURI); `resolveIcon(name)` (exact ‚Üí case-insensitive ‚Üí contains, both ways ‚Üí null).
- `components/ui/AppIcon.tsx`: `<AppIcon name size />` ‚Üí `<img>` if resolved, else monogram (first letter on `--edge`), matching the mockup. Storybook + RTL test.
- Wire in: `components/timeline/TimelineRow.tsx` (switch rows: icon replaces the bare square; keep category colour as the run spine), the run summary apps line; `views/DayView`/legend where apps are listed; Settings rule chips.

## Phase B ‚Äî Categories/rules UX (mockup `design/settings-categories.html`)

### Backend
- `db.rs`: `UnsortedApp { process_name, total_secs, last_seen }` (`specta::Type`); `get_unsorted_apps(conn) -> Result<Vec<UnsortedApp>>` ‚Äî `WHERE category_id IS NULL AND is_idle=0 GROUP BY process_name, SUM(end-start) ‚Ä¶ ORDER BY total DESC`, floor <120 s. All-time (no date args).
- `lib.rs`: `get_unsorted_apps()` command + register. Bindings regen. Repo test on a seeded DB.

### Frontend
- `components/settings/UnsortedApps.tsx`: rows (AppIcon + name + all-time secs + "showing as Other"), one-click **Assign ‚ñæ** ‚Üí `createRule(ctx,"process",name) ; reprocessLogs()`; "New category‚Ä¶" opens the editor pre-seeded.
- `components/settings/AppPicker.tsx`: search + installed-app grid (AppIcon) + selected chips; used by the category editor and the exclusion modal.
- `CategoryEditorModal.tsx`: replace free-form rule input with AppPicker; add conflict warning (first-match-wins, "Move it"), live preview (selected apps), and an **Advanced ‚Äî title pattern** drawer; copy says substring, not wildcard (`.fig` works, `*.fig` doesn't).
- `ExclusionModal.tsx`: AppPicker for the "App" match type; clearer Exclude-vs-Private copy.
- Copy pass on the section intros (plain-language "what a category is / first match wins").
- RTL tests: conflict detection, assign‚Üíreprocess, picker selection.

## Risks / watch
- `.icns` exotic formats `sips` can't read ‚Üí `None` ‚Üí monogram (acceptable). 
- Icon-map payload (~600 KB once) ‚Äî fine for one load; cached to disk for instant relaunch.
- Conflict logic drift TS‚ÜîRust ‚Äî only a warning; engine stays the source of truth.
- Virtualization must preserve the expand/collapse + "Now"/Away rows.
