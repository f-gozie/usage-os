# Phase 1.2b — the event-driven macOS capture (full native impl)

## Context

1.2a shipped the capture **seam** (`CaptureSource` trait + `Fake` + interim
`PollingCapture`). 1.2b drops in the **real, event-driven macOS impl** behind that
trait and flips `default_source()` — so the app finally captures **real window
titles, browser URLs, and terminal project signals**, event-driven (D29), instead
of polled app-names-only. This is the riskiest native code: it **compiles on macOS
CI** but can only be **runtime-verified on the Mac** (TCC grants + GUI), so the PR
ships with a documented on-device checklist. Scope (user choice): **everything** —
AX app+title, browser URL, terminal cwd + pid selection, site parsing, project
inference.

All four spikes are proven and directly portable: `spikes/ax-observer` (the model),
`spikes/browser-url`, `spikes/proc-cwd`, `spikes/project-inference`.

## Architecture changes to the 1.2a seam

**1. Channel → dedicated consumer thread (R57).** Enrichment shells `git`
(project inference) and SQLite writes block — neither may run on the async executor.
Switch the capture channel from `tokio::sync::mpsc` to **`std::sync::mpsc`** and run
the consumer on a **dedicated `std::thread`** (`rx` loop, ordered, blocking-safe).
The trait becomes `start(self: Box<Self>, tx: std::sync::mpsc::Sender<FocusEvent>)`.
`PollingCapture` → a `std::thread` sleep-loop (drops tokio); `FakeCapture` unchanged.

**2. `FocusEvent` gains `cwd: Option<String>`** (the terminal project signal;
ephemeral, so it's resolved to a project at capture time, never persisted raw).

**3. lib.rs `setup` (main thread) wiring:**
```rust
let (tx, rx) = std::sync::mpsc::channel::<FocusEvent>();
capture::default_source().start(tx);        // macOS: registers run-loop observers HERE (main thread); polling: spawns its thread
let consumer_conn = db_conn.clone();
std::thread::spawn(move || capture::consume(consumer_conn, rx));
```
The macOS source MUST start on the main thread so its observers attach to Tauri's
main `CFRunLoop` (D29). `consume` replaces 1.2a's async `run`.

## `capture/macos/` (`#[cfg(target_os = "macos")]`) — port of the spikes

- **`capture/macos/mod.rs`** — `MacosCapture` + the event-driven model, ported near
  verbatim from `spikes/ax-observer/src/main.rs` (it's hard-rule-3-clean and
  production-shaped): NSWorkspace `didActivateApplication` block observer →
  `switch_to`; per-PID `AXObserver` rebuilt on app switch; `AXFocusedWindowChanged`
  on the app element + `AXTitleChanged` re-pointed to the focused window; title
  dedupe debounce; `CallbackContext` as the `refcon`; `Drop` removes the source
  first (no UAF). `start()` registers into `CFRunLoop::current()` (the main loop)
  and **leaks the keep-alives** (`std::mem::forget` the activation token + block +
  the `Rc<RefCell<CaptureState>>`) — capture runs for the whole process, and these
  objc2 types are `!Send` so can't live in Tauri state. On each emit it builds a
  `FocusEvent`, calling the browser/terminal helpers when the active app is one.
  Trust: if `!AXIsProcessTrusted()`, prompt once (`AXIsProcessTrustedWithOptions`)
  and proceed (degrades to no-titles until granted — full priming is Phase 4/D21).
- **`capture/macos/browser.rs`** — front-tab URL, ported from `spikes/browser-url`:
  bundle-id → AppleScript app-name map (Chrome/Brave="Brave Browser"/Edge="Microsoft
  Edge"/Arc/Safari); **`mode of front window` read FIRST**, skip any non-`"normal"`
  (D8/C5); Safari safe-default = no URL unless proven non-private (R18); `-1743` →
  permanent fall back (C6); static osascript via `/usr/bin/osascript` (C7).
- **`capture/macos/terminal.rs`** — cwd, ported from `spikes/proc-cwd`
  (`proc_pidinfo(PROC_PIDVNODEPATHINFO)`), plus **front-tab shell-pid selection**
  (the spike didn't solve this): iTerm2 via osascript `current session of current
  window`'s `session.path` (reliable); generic terminals via `proc_listchildpids`
  from the terminal pid + tty/recency heuristic; abstain (cwd `None`) if
  unresolved — inference then leaves the span `unassigned` (D30), which degrades
  cleanly. **This is the least-certain piece; flagged for on-device iteration.**

## `enrich/` (cross-platform, CI-testable) — port of project-inference
- **`enrich/mod.rs`** — `parse_site(url) -> Option<String>` (host extraction);
  `infer_project(conn, &ProjectSignals{cwd,url,title}) -> ProjectAssignment` that
  applies the D30 heuristic and, on a confident signal, calls
  `db::resolve_or_create_project` (canonical_key + aliases) returning a `project_id`;
  else returns the abstain kind (`no-signal` | `ambiguous`).
- **`enrich/project.rs`** — the heuristic ported from `spikes/project-inference`:
  cwd→git remote (`git -C <cwd> rev-parse --show-toplevel` + `remote get-url origin`
  → `owner/repo`, cached per repo root); `github.com/owner/repo` extraction with the
  non-repo denylist; `file://…/projects/<name>`; window-title trailing chunk;
  abstain vocabularies (localhost/dashboards = `ambiguous`; general hosts/bare app =
  `no-signal`). Abstain threshold per D30 (assign only HIGH/MED + unambiguous).

## `capture/mod.rs` updates
`process_focus_event(conn, ev)`: compute `site = ev.url.and_then(parse_site)`; run
D8 `match_exclusion(app, title, site)` (Exclude→drop; Private→write time only, omit
title/url/site, no project); else `infer_project` → write via `db::log_focus` with
`project_id`/`project_abstain_reason`/`site`. `default_source()` →
`#[cfg(target_os="macos")] Box::new(MacosCapture::new())`, else `PollingCapture`.

compute category via `find_category`, else `insert_event`), so project_id/site/
abstain flow through. `log_activity` wrapper builds a minimal `NewEvent`
(`..Default::default()`) — existing coalescing tests unchanged.

  runtime-verified on-device (checklist below). No unit tests assert AX behavior.

## On-device verification (the manual gate — owner: Favour, in the PR description)
`cargo tauri dev`, grant Accessibility (+ Automation per browser when prompted):
1. Switch apps → spans recorded with **real titles** (editors, Chromium/Electron).
2. Change a window title in place → a new coalesced span (AXTitleChanged).
3. A browser normal tab → URL + site captured; **incognito/private → NO url** (D8).
4. A terminal in a git repo → the span gets the right `project_id`; `/` → unassigned.
5. Idle ~3 min → idle spans; `powermetrics` near-zero wakeups while untouched (R11).
6. Deny a browser's Automation → that browser falls back, others keep working (C6).
